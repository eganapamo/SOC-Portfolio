# Lab 1 – Basic Log Search Queries

## Objective
Understand how to search logs in Splunk, filter events, extract fields using regex (rex), and analyze authentication activity.

Environment:
- Splunk Enterprise (Localhost)
- Metasploitable 2 (192.168.56.102) – Log source
- Kali Linux (192.168.56.104) – SSH attacker


## Step 1 – Searching for Failed SSH Attempts

Search query:

index=main "failed password"

Purpose:
Identify failed authentication attempts in SSH logs.

Result:
Metasploitable (192.168.56.102) logged failed login attempts from Kali.


## Step 2 – Extract Attacker IP Using rex

Search query:

index=main "failed password"
| rex "from(?<attacker_ip>\d+\.\d+\.\d+\.\d+)"
| table _time host attacker_ip

Explanation:
- Regex extracts IPv4 addresses
- Table displays time, log source host, and attacker IP

Result:
Host: 192.168.56.102 
Attacker IP: 192.168.56.104 

Key Lesson:
Splunk only shows what the log source machine records. The attacker IP appears because Metasploitable logs incoming SSH attempts.


## Step 3 – Filtering by Host

Search query:

index=main host="192.168.56.102"

Purpose:
Filter logs to only events generated by Metasploitable.

Lesson:
"host" field represents the system generating the log, not the attacker.


## Step 4 – Simulating Privilege Escalation & Command Activity

Executed on Metasploitable:

sudo su
sudo chmod 777 /etc/passwd
adduser labuser

Search query used to identify suspicious commands:

index=main host="192.168.56.102"
("sudo" OR "chmod" OR "adduser" OR "usermod" OR "groupadd" OR "bash")

Purpose:
Identify potential privilege escalation, persistence, or log tampering behavior.


## Key Concepts Learned

1. Difference between:
   - host (log source)
   - attacker_ip (source of connection)
   - Splunk server (log storage)


2. Splunk only sees what systems log.
   - No connection = no event
   - No log ingestion = no visibility

3. Basic filtering and time range narrowing for investigation.


## Conclusion

Lab 1 reinforced foundational Splunk skills:
- Searching
- Filtering
- Field extraction with rex
- Understanding log origin and data flow
- Identifying authentication failures and suspicious command usage

Time scoping significantly affects visible telemetry. Narrowing the investigation window to 30 minutes post-authentication revealed immediate privilege escalation activity, while expanding the window to full-day scope revealed user creation activity (persistence).

This lab establishes the foundation for detection engineering and behavioral analysis in future labs.
